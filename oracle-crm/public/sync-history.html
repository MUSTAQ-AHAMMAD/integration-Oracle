Troubleshooting Data Sync Issues

Common reasons why data might not be syncing to Oracle Fusion, and how to fix them:

SymptomLikely CauseFix
Invoice not created Missing Oracle credentials or wrong base URL Go to Configuration, verify FUSION_BASE_URL and credentials
API error 401 Wrong username or password Reset Oracle Fusion password, update .env file
API error 400 on invoice Missing required fields (BillToName, TxnSource, TxnType) Fill in Section B – Oracle Fusion Metadata on the push form
Duplicate invoice line error Sale already pushed – duplicate detection triggered Check Oracle Fusion for existing transaction number
Wrong GL date in Oracle Timezone offset not configured Set correct timezone offset in the form (e.g. 4 for UAE UTC+4)
Receipt not applied ReceiptMethodId or BankAccountId not set Configure receipt method metadata in the push form's Payment section
Inventory transaction fails Organization not found in Oracle Verify outletName matches Oracle Organization code
Journal not created CustomerType = NORMAL (journals skipped by design) Only non-NORMAL customer types create journal entries
