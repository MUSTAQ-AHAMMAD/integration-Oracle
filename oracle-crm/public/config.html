How to Configure Credentials
Security: Credentials are stored in a .env file on the server — never commit this file to Git.
Step 1 – Create the .env file

In the oracle-crm/ directory, create a file named .env:

# ── Oracle Fusion Cloud ────────────────────────────────────────────── # URL pattern: https://{hostname}.fa.{region}.oraclecloud.com # Example: https://ehxk-test.fa.em2.oraclecloud.com FUSION_BASE_URL=https://your-instance.fa.em2.oraclecloud.com FUSION_USERNAME=your.oracle.username FUSION_PASSWORD=your_oracle_password # ── Odoo ERP Connection ─────────────────────────────────────────────── # All Odoo data is fetched via JSON-RPC at {ODOO_URL}/jsonrpc ODOO_URL=https://your-odoo-instance.odoo.com ODOO_DB=your-odoo-database-name ODOO_USERNAME=your-odoo-username ODOO_PASSWORD=your-odoo-password-or-api-key # ── Optional settings ───────────────────────────────────────────────── PORT=3000 REGIONS=AE,KW,OM,SA,BH,QA LOG_LEVEL=info
Step 2 – Oracle Fusion URL structure
https://{hostname}.fa.{region}.oraclecloud.com Examples: https://ehxk-test.fa.em2.oraclecloud.com ← Europe Middle East 2 (em2) https://acme-prod.fa.us2.oraclecloud.com ← US West (us2) https://myco.fa.uk2.oraclecloud.com ← UK (uk2) The REST API path appended to this base URL is: /fscmRestApi/resources/11.13.18.05/{endpoint}
Step 3 – Odoo connection
# All Odoo operations use a single JSON-RPC endpoint: # POST {ODOO_URL}/jsonrpc # # The service/method/model in the request body selects the operation: # common.authenticate – get uid (called once per job) # object.execute_kw – search_read on sale.order, sale.order.line, stock.warehouse # # The Odoo username must have access to: # - sale.order (Sales / read) # - sale.order.line (Sales / read) # - stock.warehouse (Inventory / read)
Step 4 – Restart and verify
cd oracle-crm npm install npm start # Open http://localhost:3000/config.html # Use the "Test Connection" buttons to verify both systems
Credentials are validated live by the "Test Connection" buttons above and again on each push job — not at server startup.
