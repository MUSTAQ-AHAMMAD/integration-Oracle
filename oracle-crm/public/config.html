How to Configure Oracle Credentials
Security: Credentials are stored in a .env file on the server — never commit this file to Git.
Step 1 – Create the .env file

In the oracle-crm/ directory, create a file named .env:

FUSION_BASE_URL=https://your-instance.fa.em2.oraclecloud.com FUSION_USERNAME=your.oracle.username FUSION_PASSWORD=your_oracle_password PORT=3000 REGIONS=AE,KW,OM,SA,BH,QA
Step 2 – Find your Oracle Fusion base URL

Your Oracle Fusion Cloud URL has this structure:

https://{hostname}.fa.{region}.oraclecloud.com Examples: https://ehxk-test.fa.em2.oraclecloud.com ← Europe Middle East 2 https://acme-prod.fa.us2.oraclecloud.com ← US 2 https://myco.fa.uk2.oraclecloud.com ← UK 2
Step 3 – Start the CRM server
cd oracle-crm npm install npm start # Server will start on http://localhost:3000
Step 4 – Verify the connection

After saving your .env and restarting the server, reload this page. The status above will show "Configured" if credentials are set.

Note: The CRM validates credentials on each API call. An API test call to Oracle Fusion is only made when you perform a sale push — not on startup.
